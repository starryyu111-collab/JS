\section{Experiments}
\label{sec:experiments}

\subsection{Dataset and Model}
\label{subsec:dataset_model}

The experiments use CIFAR-10, a standard image classification dataset with ten object categories \cite{krizhevsky2009learning}. The model is a CIFAR-style ResNet-18, denoted as \texttt{resnet18\_cifar}. It follows the residual-network design principle of using shortcut connections to support deeper convolutional models \cite{he2016deep}, while being adapted to the small-image CIFAR setting. A trained FP32 checkpoint is used as the starting point, and all quantized variants are produced from this fixed model. Accuracy is evaluated on the full CIFAR-10 test set.

\subsection{Calibration Protocol}
\label{subsec:calibration_protocol}

Calibration uses 1024 CIFAR-10 training samples drawn with seed 42 and a mini-batch size of 128. The calibration subset is sampled only from the training split and is used only to collect activation statistics and determine clipping thresholds. It is not used for reporting classification accuracy, and no model weights are updated during calibration. This separation keeps calibration distinct from test-set evaluation.

For INT4-MSE-Selected, calibration activations are collected at the observed post-ReLU sites. Each site uses the same candidate percentile set, but the selected threshold can differ across sites because activation distributions vary across the network. For INT4-MinMax and INT4-P99.9, calibration also uses the training subset, but the threshold rule is fixed rather than selected by local reconstruction MSE.

\subsection{Compared Quantization Settings}
\label{subsec:compared_settings}

This work evaluates simulated PTQ with fake quantization and dequantization in floating-point computation. No real INT4 hardware kernel or latency measurement is included. The compared settings are FP32, INT8-MinMax, INT4-MinMax, INT4-P99.9, and INT4-MSE-Selected. INT8-MinMax uses signed symmetric per-channel weights in $[-127,127]$ and uint8 affine activations in $[0,255]$. The INT4 variants use signed symmetric per-channel weights in $[-7,7]$ and post-ReLU uint4 activations in $[0,15]$. For INT4 post-ReLU activations, the affine zero-point is fixed to zero because the calibrated activation interval is $[0,\alpha_l]$. Because INT8-MinMax and the INT4 variants use different activation sites and granularities, INT8-MinMax serves mainly as a sanity-check reference baseline. The controlled calibration comparison is among INT4-MinMax, INT4-P99.9, and INT4-MSE-Selected.

\subsection{Metrics}
\label{subsec:metrics}

The primary metric is Top-1 accuracy on the CIFAR-10 test set. Accuracy drop is reported relative to the FP32 model to show the degradation introduced by quantization. Activation reconstruction MSE is computed between FP32 post-ReLU activations and their fake-quantized/dequantized counterparts. The same reconstruction-error form is used during calibration for threshold selection, while the reported activation MSE is computed for evaluation and comparison. Logit MSE compares FP32 and quantized model logits on the evaluation data. It measures output-level consistency with the FP32 model, but it does not directly count classification decisions. These error metrics characterize simulated quantization behavior and are interpreted together with accuracy, rather than as replacements for accuracy.
