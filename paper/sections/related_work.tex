\section{Related Work}
\label{sec:related_work}

\subsection{Post-training quantization}
Post-training quantization (PTQ) compresses a trained floating-point neural network by determining quantization parameters from a small calibration set, without updating the model weights through back-propagation. Compared with quantization-aware training (QAT), PTQ is easier to integrate into an existing training workflow because it does not require retraining or modifying the original optimization process. In a typical PTQ pipeline, weights are quantized using statistics from the trained parameters, while activation ranges are estimated from calibration data. The quality of these estimated ranges is critical because the same fixed quantization parameters are then used during inference. This makes calibration a central component of PTQ, especially when the target activation bit-width is low.

\subsection{Low-bit quantization for CNNs}
CNNs are often deployed under memory and computation constraints, which motivates low-bit weight and activation quantization. INT8 quantization is commonly used in practical inference pipelines because it usually provides a favorable trade-off between compression and accuracy. However, reducing precision further to INT4 is more difficult. With only 16 activation levels, the quantization interval becomes much coarser, and errors caused by activation range estimation can be amplified. This is particularly important for CNNs with ReLU activations, where most activation values may be concentrated near zero while a small number of large values determine the MinMax range.

Existing low-bit CNN quantization methods address this problem from different directions, including improved calibration, mixed-precision assignment, clipping, reconstruction-based optimization, and QAT. This work focuses on a narrower setting: simulated INT4 PTQ for a CIFAR-10 ResNet-18-CIFAR model. It does not attempt to compete with full QAT methods or large-scale low-bit deployment systems. Instead, it studies whether a simple calibration-time activation clipping rule can make an INT4 PTQ baseline less sensitive to activation range selection.

\subsection{Activation clipping and outlier suppression}
Activation clipping is a common way to reduce the influence of rare large activation values during quantization. Under MinMax calibration, the maximum observed calibration activation determines the upper bound of the quantization range. If this maximum comes from an outlier, the resulting step size may become too large for the dense region of the activation distribution, increasing rounding error for many samples. Clipping lowers the upper bound and may reduce rounding error, but it also saturates values above the threshold. Therefore, the clipping threshold controls a trade-off between rounding error and clipping error.

Different clipping strategies can be used in PTQ. A fixed percentile threshold is simple and often effective, but it applies the same rule regardless of layer sensitivity. Reconstruction-driven calibration chooses thresholds by minimizing a proxy error, such as the mean squared error between original activations and fake-quantized--dequantized activations. The method studied in this paper follows this idea in a deliberately simple form. For each post-ReLU activation site, it evaluates a predefined percentile candidate set and selects the candidate with the smallest activation reconstruction MSE on calibration samples. This is a layer-wise candidate selection procedure, not a continuous-space global optimization method.
